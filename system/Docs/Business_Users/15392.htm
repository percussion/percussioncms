Moving Active Assembly Items Across Sites

In an Active Assembly relationship, the dependent item's Site and Folder may be properties in the Relationship. If the Site and Folder are properties of the Relationship, a user cannot use the normal Move action to move the dependent item in the Relationship from its original Site to another Site because its Active Assembly link will be broken. Likewise, a user cannot use the normal Remove action to remove the dependent item from its original Site.

Rhythmyx provides special Force Move and Force Remove actions, which reset the Site and Folder properties and the link addresses. By default, these actions are only assigned to administrators in administrator Communities.

Rhythmyx displays an Error Message if a user attempts to move a dependent to another Site or to remove a dependent whose Site and Folder are properties in an Active Assembly relationship:

If the Site is a property of the Relationship, but the Folder is not, the behavior is similar. However, if the dependent item appears in more than one Folder on the Site, a user can perform a common Move or Remove action on the item until the dependent item only appears once on the Site. At that point, a Move or Remove action triggers the dialog shown above and an administrator must use a Force Move or Force Remove action to move/remove the item and reset the links.

A user can move a dependent whose Site (and Folder) are properties of an Active Assembly Relationship from a Site Folder to a non-Site Folder. The original Site remains a property of the Relationship and the Folder is updated. For details of how Site and Folder properties of Relationships change when a dependent item is moved between different types of Folders, see the document Implementing the Relationship Engine.

By default, the FastForward sample Active Assembly Relationships do not include Site and Folder properties. Site and Folder properties are included in Active Assembly Relationships when they are displayed in the search results.
