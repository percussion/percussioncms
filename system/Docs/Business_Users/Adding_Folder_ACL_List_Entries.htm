Adding Folder ACL List Entries

An Access Control List (ACL) describes each possible set of users and their respective access levels for a system component such as an application or server. In Content Explorer, each folder has an ACL that describes the possible sets of folder users and their level of access to the folder.

If you have Admin permission for a folder, from the Security tab of its Folder Properties dialog, you may access the Folder ACL List Entry Editor to view or modify the Roles, users, and other groups that have access to the Folder. The Cataloged Entries list displays available entries that you have not added to the Folder ACL. The Current ACL Entries list displays entries in the Folder ACL List.

The Folder ACL List Entry Editor does not determine an entry's level of access to a Folder; the level of access is indicated in the Security tab of Folder Properties dialog.

To add Folder ACL List entries:

  1. In the Folder Properties dialog click [Add].

    Rhythmyx opens the Folder ACL List Entry Editor.

  2. Under Show Catalog of choose a category of Entries:
    • Roles - When initially selected, the Cataloged Entries list displays all Roles known to Rhythmyx Server.
    • Users - When initially selected, the Cataloged Entries list displays users in all Roles known to Rhythmyx Server. NOTE: Since managing User entries in the ACL List can be cumbersome, only include individual users if necessary. For example, you may be required to add a user who logs on through the single sign-on feature and is not included in any cataloged entries for users or Roles.
    • Virtual - Displays the groups Everyone and Folder Community. Everyone includes all users logged into the system. Folder Community includes all members of the folder creator's Community. You may remove these entries but may not add additional Virtual entries.

    NOTES: A User's settings supersede those of Roles or Virtual entries that include the user. If a user is a member of more than one Role or Virtual Entry (and not in a User entry) the maximum access in any Role or Virtual Entry is applied.

  3. In Cataloged Entries select an entry and click [Add>>] to move it to the Current ACL Entries list.

    By default, new entries have Read permission.

  4. In Current ACL Entries, select an entry and choose [<<Remove] to move it from the Current ACL Entries list to the Cataloged Entries list.
  5. To create a new entry in the Cataloged Entries list, click [New].

    Rhythmyx opens the New User ACL Entry dialog.

    NOTE: In general, use this dialog to add individual members rather than Roles to a Folder ACL. It is used most frequently in systems that use single sign-on, since Rhythmyx cannot catalog single sign-on users.

  6. In the Security Provider Type drop list, choose a Security Provider. The drop list displays all security provider types known to the Rhythmyx Server.
  7. In the Provider Instance drop list, choose a provider instance for the security provider type. The drop list displays all provider instances known to the Rhythmyx Server for the Security Provider Type chosen.
  8. In Name enter a name for the entry.
  9. Click [OK].

    Rhythmyx adds the entry to the Current ACL Entries List.

  10. When you have finished modifying the Folder ACL List, click [OK] to close the dialog and return to the Security tab of the Folder Properties dialog.

The first time you view Roles, Users, Security provider types, and Provider instances in the ACL List Entry Editor, the Editor retrieves them from the Rhythmyx server and caches them so it does not have to retrieve them from the Rhythmyx Server when you open the Editor again. Therefore, after you initially open the ACL List Entry Editor if you want changes in the Rhythmyx Server reflected in it, you must restart your Rhythmyx session.

Users who access the system using the Single Sign-On feature do not appear in the Cataloged Entries list, and you must add them manually with the New User ACL Entry dialog.
