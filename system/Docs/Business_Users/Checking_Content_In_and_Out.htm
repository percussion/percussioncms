Checking Content In and Out

Rhythmyx includes a check-in and check-out feature that allows multiple users to access a Content Item but prevents more than one user at a time from actively editing an item.  When you check out a Content Item, you make it available to you to edit, and make it unavailable to all other users.  When you have finished working with the Content Item and have updated your changes to the database, check the item back in to make it available to other users.  

Check-in and check-out are special types of Transitions that do not move a Content Item from one State to another, but change their accessibility by setting a lock flag that indicates whether a user has checked the item out, and which user has checked the item out. Users cannot check-in or check-out items that are in a public State. Check-in and check-out are available from the Workflow > Transition sub-menu of the Action Menu. 

While a Content Item is checked out:

  • no other user can check out that Content Item;
  • only the user who has that Content Item checked out can edit it;
  • only the user who has that Content Item checked out can create a new Copy of it;
  • no other user can perform a workflow action on the Content Item, except the Administrator, who can perform a forced check in.

While a Content Item is checked in:

  • any user who has access to the Content Item can check it out;
  • any user who has access can perform a workflow transition on the Content Item.

Some of the actions available in the Action Menu cause Rhythmyx to automatically check items in and out for you. When you select Edit in the Action Menu, if the Content Item is not already checked out, Rhythmyx checks it out to you.  When you perform a Workflow transition on a Content Item from the Action Menu, if the item is not already checked in, Rhythmyx checks it in before performing the Workflow transition.
