Revisions, Copies, and Versions

A Revision is a modification to an existing Content Item after it has been published. It has a separate Revision ID from the original item, but the same Content ID. A Copy is a duplicate of a Content Item with a new Content ID. While a Revision becomes the current Content Item, a Copy becomes an entirely new Content Item and moves through Workflow independently of the original. A Version, like a copy, is a duplicate of a Content Item with a new Content ID, but when a Version reaches a public State, the Content Item that it copied is transitioned into an archive State. Rhythmyx sometimes refers to Versions as Promotable Versions because Versions are always promoted when they reach a public state. Copies and Versions link to the same related Content Items as the original Content Items; Rhythmyx does not create duplicate related Content Items for them as it does for Translation Copies.

Rhythmyx stores the owner/dependency Relationships between Content Items and their Copies or Versions. You can view these Relationships in the Impact Analysis window for the Content Item. You can view all Revisions of a Content Item in the Revision List. Through the Revision List, you can promote an earlier revision (make it the current revision).

Rhythmyx creates revisions automatically when you modify and save published content. You create a Copy through the Create > New Copy Action Menu option, and a Version through the Create > Promotable Version Action Menu option. In general, Copies are useful for creating new Content Items that are only slightly different from existing Content Items. Versions are useful when you want to change a published Content Item, but not publish the changes yet. They also allow you to save the original Content Item in case you want to republish it in the future.

NOTE: Workflow transitions for all revisions of a Content Item are displayed in the Audit Trail.
