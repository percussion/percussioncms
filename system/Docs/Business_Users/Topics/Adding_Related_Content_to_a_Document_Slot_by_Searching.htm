Adding Related Content to a Document Slot by Searching

To search for and add related content to a Slot in the Document Assembly Window Navigation Tree:

  1. Right-click the Slot node , and select Search.

    OR

    Select the Slot node , and in the Menu Bar, choose Content > [Slot Type] > Search.

    Rhythmyx displays the Content Search dialog for Slots.

  2. Enter search parameters. See Basic Search for help.

    The search for Slots differs in some ways from the basic search:

    • The search only searches for Content Types allowed in the Slot. If you choose a Custom Search Category, only allowed Content Types are available in the drop list, and the results of other Categories are filtered by allowed Content Types.
    • Only Display Formats that include Content Type and Template are available in the Display Formats drop list.
  3. When you finish entering the fields in the Content Search dialog, click [OK].

    The dialog closes, and Rhythmyx runs the search. In the Document Assembly window, Rhythmyx highlights the selected Slot and displays Category icons for the Template types returned in the Display pane.

  4. Expand the category icon of the Template that you want to add to the Slot.

    Rhythmyx displays the Content Items returned by the Search for the Template. Depending on how your Content Types are configured, the icon preceding each content item in the right pane may be the standard content item icon (see below), or may be a custom icon based on the Content Type.

  5. In the Display pane, right-click the Content Item that you want to insert in the Slot.

    Rhythmyx displays a drop menu.

  6. In the drop menu, choose:
    • Link to Slot to inserts the selected Content Item in the Slot in the Navigation Tree;
    • Preview to view the the assembled Template;
    • Copy, then right-click any Slot that accepts the Content Type and Template and choose Paste to insert the Content Item into the Slot.

    Rhythmyx saves the search query in the Navigation Tree under Search Results in the New Search node. If the New Search node already contains a search query, the new search query overwrites it.

You can change the Display Format of a Document Assembly window search as you can for a Content Explorer search. See Display Formats for help.

In the Document Assembly window Navigation Tree, you cannot save searches as you do in the Content Explorer Navigation Tree. You can only re-access the most recent search. A new search always overwrites the previous one.
