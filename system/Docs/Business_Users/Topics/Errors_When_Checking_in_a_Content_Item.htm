Errors When Checking a Content Item In

Rhythmyx checks the most recent actions performed on a Content Item prior to displaying available Action Menu options to avoid displaying Workflow options that have already been performed. However, on occasion two users may simultaneously view Check In or Check Out on Action Menus for the same item and may both attempt to check in or check out the item. If this occurs, the user whose action reaches the server second receives an error message because the first user already completed the same action.

If you attempt to check in or check out a Content Item and receive an error message, redisplay the Action Menu to see if the item has already been checked in or checked out by another user.
