Folder Properties Dialog

The options New Folder and Properties open the Folder Properties dialog. If you choose New Folder, the dialog fields are empty or set at default values and many are enabled for editing. If you choose Properties, the fields are set for the selected folder and the fields enabled for editing depend on your folder security level.

Below, the Folder Properties dialog is opened to the General tab for a new folder, so its fields are empty or set at default values. The general properties include basic information that defines the folder, such as its name and its Community

General Tab Fields:

Folder Name - Name of the folder shown in the Navigation Tree.

Folder ID - ID number given to the folder object. Non-editable.

Folder Community - The Community whose members have access to the folder. All communities indicates all users in all Communities.

Location - The location of the folder in the Navigation Tree. Non-editable.

Description - Optional description of the folder.

Locale - Locale assigned to the folder object.

Default display format - Display Format applied to the folder.

Global Template Specifies the Global Template to use when assembling Content Items in the Folder. Options include all Global Templates set up in the system. If you specify a Global Template for the Folder, it will override the default Global Template for the Site when assembling Content Items from that Folder. This field is only available on Folders that are nodes of the Sites root.

Publish Only in Special Edition Checkbox. Outdated feature formerly used for special processing.

The Security tab lists the Folder Access Control List (ACL), which includes all members who have access to the folder. When you select a member of the list, the checkboxes to the right display the member's specific access. If none of the fields are checked for a member, the member cannot view or edit the folder.

Security Tab Fields:

Names - The Folder ACL List (Users, Roles, and other groups of users who may have access to the folder). If you hover your cursor over a name, a pop-up displays Security ACL Type:Provider instance//Name.

Permissions - Types of access the selected Name has to the folder. Choices are Read, Write, and Admin. Any combination or none of these permissions may be checked.

[Add] - Access the ACL List Entry Editor dialog to add a Name.

[Remove] - Remove the selected Name from the Names list.

The Properties tab displays custom properties defined for folders in your CMS. Usually, your System Administrator defines these fields, and you should not attempt to modify them unless you are instructed.

Custom Tab Fields:

Name - Name of custom Folder property. Type in a custom property name or click in the Name cell to access a drop list that gives you the name of the only Rhythmyx-defined Folder property, sys_pubFileName. sys_pubFileName lets you specify a name the Folder will have on the Site that is different than the name it has in Content Explorer.

Value - Value of custom Folder property.

Description - Description of custom Folder property.

Note: For information about the access and permissions required to perform different actions on folders and items in folders, see the Security Concepts page at help.percussion.com/cmsystem.
