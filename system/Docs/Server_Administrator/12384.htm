Maintaining Caching Settings

The Rhythmyx server includes a cache that stores the following

  • assembled pages
  • query result pages
  • Folder contents and navigation.

The cache improves the performance of your CMS by preserving data for quick recall later.

Use the Caching subtab of the Settings tab to maintain caching settings.

The Enable Page Cache checkbox controls caching of assembled pages and of query results pages. The default value of this box is checked. If you want to disable page caching, uncheck this box.

If you enable page caching, you can define the caching parameters:

  • The Maximum Amount of Memory (MB) field defines the maximum amount of memory the server can use for page caching. The default value of this field is 100 MB. If you want to allow unlimited space for caching, check the Unlimited checkbox for this field.
  • The Maximum Amount of Disk Space (MB) field defines the maximum amount of disk space that the server can use for caching. The default is 1042 MB. To allow the server to use an unlimited amount of disk space for caching, check the Unlimited checkbox.
  • The Maximum Page Size (KB) field defines the maximum size of a page that may be cached. he default is 100 kilobytes. To allow the server to cache pages of any size, check the Unlimited checkbox.
  • The Aging Time (min) field defines the maximum amount of time that an item may remain in the cache before the server flushes it. To allow items to remain in the cache an unlimited amount of time, check the Unlimited checkbox. The default is unlimited.

The Enable Folder Cache checkbox controls caching of Folder contents and navigation. The default value of this box is checked. If you want to disable Folder caching, uncheck the box. (Note that if Folder caching is enabled, you must attach the sys_FlushCache post-exit to Content List resources to ensure that the cache is flushed during Publishing. Also, you should use the rxs_SiteFolderContentListBulkBuilder exit when implementing Site Folder Publishing rather than using the rxs_SiteFolderContentListBuilder exit.
