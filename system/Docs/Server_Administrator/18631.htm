Logging Tab, View Sub-tab

You can dynamically retrieve XML application log entries using the Logging tab's View sub-tab. The View sub-tab lets you choose the types of log entries you want to view and the date range of entries to include.

Query conditions/Type

Errors – Show errors that occurred during processing.

Server start – Show when the Rhythmyx Server started running.

Server stop – Show when the Rhythmyx Server stopped running.

Application start – Show when each application started running.

Application stop – Show when each application stopped running.

Application statistics – Show the time an application is active, events it processes, and performance statistics.

Basic user activity – Show common user activities such as session id, username, URLs for each request, and the host address of the requestor.

Detailed user activity – Show basic user activities as well as the input XML document, HTML parameters, and request statistics.

Full user activity – Show basic and detailed user activity and any additional information encountered while processing the request, such as completed SQL statements.

Warnings – Show potentially harmful events that do not cause the Server to stop running.

Multiple app. Handlers - For multiple request handlers, show the message for each resource that matches the request criteria. The first resource that matches the request criteria processes it, but Rhythmyx checks subsequent resources. Use only for debugging purposes.

Query conditions/Date

Start – The start date of log entries to retrieve. Must be in the format yyyymmdd hh:mm:ss. You can leave the Start date empty to retrieve entries from the beginning of the log up to the End date.

End – The end date of log entries to retrieve. Must be in the format yyyymmdd hh:mm:ss. You can leave the End date empty to retrieve entries from the Start date through the end of the log.

NOTE: If you leave both the Start and End dates empty, you retrieve all entries in the log that match your other conditions.

[Get It] – Clicking this button sends the query and returns the requested entries in a new window.
