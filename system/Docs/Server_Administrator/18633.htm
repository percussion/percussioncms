Logging Tab, Settings Sub-tab

You can configure which statistics to regularly store in the XML application log using the Logging tab's Settings sub-tab. This sub-tab lets you choose the types of log entries you want to record and how long you want to store log entries.

Keep Log of

Errors - Log errors that occur during processing. Checked by default.

Server startup and shutdown – Log entries when the Rhythmyx Server starts and stops running.

Application startup and shutdown – Log entries when each application starts and stops running.

Application statistics at shutdown - Log performance and processing statistics for an application.

Basic user activity – Log common user activities such as session id, username, URLs for each request, and the host address of the requestor.

Detailed user activity - Log basic user activities as well as the input XML document, HTML parameters, and request statistics. Only enable when you are actively debugging; output is verbose.

Full user activity - Log basic and detailed user activity and any additional information encountered while processing the request, such as completed SQL statements. Only enable when you are actively debugging; output is verbose.

Auto Delete

Auto Delete – When checked, automatically deletes logs at the intervals specified in Keep last [nnn] days of Log. Checked by default.

Keep last [nnn] days of Log – Enter the number of days the Server should store a log entry before deleting it. 2 by default.
