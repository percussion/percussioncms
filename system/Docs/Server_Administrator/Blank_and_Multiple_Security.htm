Blank and Multiple Security

You can choose not to specify a security provider type by choosing <any> in the security provider list in any security provider field.

Another option is <any> type.  If you choose this option, once you define instances of that security provider type, the server can authenticate with any instance of that type.  For example, if you choose <any> Database Directory, the server can authenticate any user with security providers of the type Database Directory.  You can also define several providers of the same type.  For example, you could define a Database Directory security provider for Oracle and another Database Directory security provider for MS SQL Server.
