Optimizing performance

When you install the Rhythmyx server, Settings parameters are set to default values. Settings for optimal performance are system-dependent and you can fine-tune the settings to increase your system's speed and efficiency.

Settings are divided into two categories:

  • Performance - to control threads, queue depth, and time out settings for servers and applications. You can also specify whether user session parameters override application settings. If the browser or other tool you use to generate requests to the Rhythmyx server does not support cookies, uncheck "Session Management".
  • Caching - to enable or disable assembler and resource page caching by the server, to set caching maximum and timeout settings. Assembler page caching saves assembled pages so that Rhythmyx does not have to use system time rolling up the slots and items every time the page is accessed. Resource page caching saves the results of query resources so Rhythmyx does not have to repeatedly run the same query during a session. NOTE: Rhythmyx does not execute exits on any query resources when it retrieves cached pages.
