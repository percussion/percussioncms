About the Root Node

All XSpLit fields share a common root node. The root node is the base nesting level of hierarchically structured XML documents.  In XSpLit, the root node takes the name of the HTML document that you split.  All nodes created by XSpLit share this root node, which associates the fields with the parent document, automatically.   (The root node is a requirement of XML documents.) Since the root node is automatically part of all field names and designers don't usually enter it, the repeating rules topics don't take the root node into consideration.
