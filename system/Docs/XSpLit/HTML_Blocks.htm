HTML Blocks

Common practice is to compose HTML pages of different blocks of islands of reusable text. For example, a header or copyright information might be the same across and entire site, and would be stored as blocks of HTML code called by each page. This HTML code will contain special characters or entities, such as "&amp" for an ampersand or "&lt" for a less-than sign. By default, the XSL processor does not resolve these entities. When processing HTML blocks, however, it should resolve them.

To tell the XSL processor to resolve HTML blocks correctly, XSpLit uses the special attribute
no-escaping="yes".

XSpLit produces XSL with a default template that disables output escaping when the no-escaping attribute is set to "yes". Disabling output escaping enables the XSLT to resolve the special characters and entities used in HTML code.

 <xsl:template match="*">

    <xsl:choose>

      <xsl:when test="text()">

        <xsl:choose>

          <xsl:when test="@no-escaping">

<xsl:value-of select="." disable-output-escaping="yes"/>

          </xsl:when>

          <xsl:otherwise>

            <xsl:value-of select="."/>

          </xsl:otherwise>

        </xsl:choose>

      </xsl:when>

      <xsl:otherwise>&nbsp;</xsl:otherwise>

    </xsl:choose>

    <xsl:if test="not(position()=last())">

      <br/>

    </xsl:if>

  </xsl:template>

To add the no-escaping attribute:

  1. Open the application in the Workbench and view the XML resource properties.
  2. Open the mapper.
  3. In the XML column, add an additional entry for the attribute (fieldname/@no-escaping).  
  4. In the Back-end column map the field to a Literal set to yes.
