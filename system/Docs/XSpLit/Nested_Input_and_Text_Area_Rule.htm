Nested Input and Text Area Rule

About the Root Node

HTML does not support nesting Tables and Lists inside the form Input tag. You can nest the Input tag or an entire Form within a Table cell or List item. See Extended Table Nesting Rule or Extended List Nesting Rule for more information on how a repeating form Input element affects repeat behavior when nested in table cells or lists.

Suppose you had the fields psx-product/name, psx-product/feature, and psx-product/comment in the following HTML code:

<table border="1" width="60%">
<tr bgcolor="lightgrey">
<th>Product</th>
<th>Select Your Favorite Features</th>
<th>Comment</th>
</tr>
<tr>
<td>psx-product/name</td>
<td>
<form method="POST">
<table><tr><td>
<input type="checkbox" name="C1" value="ON">psx-product/feature
</td></tr></table>
</form>
</td>
<td>
<form method="POST">
<textarea rows="2" name="S1" cols="20">psx-product/comment</textarea>
</form>
</td>
</tr>
</table>

The three fields share the node product. The table row is the outermost repeating formatting, so it repeats once for each occurrence of product. The second column includes an Input type checkbox element. For each row, the Input element repeats once for each occurrence of feature. The last cell includes a Textarea element. For each row, it repeats once for each occurrence of comment.
