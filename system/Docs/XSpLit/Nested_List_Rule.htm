Nested List Rule

About the Root Node

The outermost list repeats for the first shared node (going left to right), the second list inward repeats for the second shared node, and so on. This behavior continues until all nested lists or shared nodes are exhausted.

So if you had the fields psx-product/name and psx-product/feature in the following code:

<ul>
<li>psx-product/name
<ul type="circle">
<li>psx-product/feature</li>
</ul>
</li>
</ul>

XSpLit processes the outer list first. The list repeats according to the basic rule, once per product.

XSpLit then processes the inner list. It will repeat once per feature.
