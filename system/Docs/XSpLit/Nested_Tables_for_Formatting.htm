Nested Tables for Formatting

Web designers often use tables for layout purposes for an entire page. Adding extra nodes to field names helps you control which table rows on the page should repeat and which shouldn't.

Suppose you had two dynamic fields, both sharing a node: psx-product/name and psx-product/feature.

XSpLit processes the outermost table first (the one used primarily to format the page). Since we don't want this table row to repeat, but it will have dynamic content in it, we add a node, productlist to the two repeating fields in the inner table. The resulting fields are named psx-productlist/product/name and psx-productlist/product/feature. The outer table row repeats only once per occurrence of productlist. Based on our data, we're certain to only have one productlist per page. This extra node effectively "uses up" the outer table.

The next table inward is the one in the second column. This table row now repeats exactly as described in Repeating Multiple Nodes.

Tip: Add extra nodes to the left of your field names to fully control how all nested table rows repeat. Remember, XSpLit makes it easy to map these fields to data no matter what names you choose.

The following HTML code shows the use of this technique.

<table border="1" width="50%">
<tr>
<td bgcolor="lightgrey" valign="top">
<b>Home</b><br /><br />
<b>Products</b><br /><br />
<b>Services</b><br /><br />
<b>etc.</b>
</td>
<td>
<p align="center"><b>Welcome to OurSite.com</b></p>
<p align="center">We've got lot's of great pages on our site, plus some dynamic content,
right here on this page too!</p>
<table border="1" width="100%">
<tr>
<td>psx-productlist/product/name</td>
<td>psx-productlist/product/feature</td>
</tr>
</table>
<p>Take a look today!</p>
</td>
</tr>
</table>
