Best Practices for Writing Manual Templates

Best practice when writing a Slot template is to use the same structure as when defining local formatting for the Slot:

  • The Slot as a whole, which defines the border and background on the Slot, as well as any heading for it. This formatting will not repeat in the Slot.
  • The Snippet wrapper, which defines the immediate formatting around each Content Item in the Slot and which repeats with each Content Item in the Slot.
  • The local content of the Snippet.

A Slot template has three requirements:

  • The value of mode for the Slot template as a whole must be rxslot. For example:

    <xsl:template match="rxslot[@template='RelatedTemplateSlot']" mode="rxslot">

  • The value of the template attribute of your manual template must match the value of template for the template you are overriding. This value is derived from the template attribute used when defining the Slot location for XSpLit.
  • The value of mode for the Snippet wrapper inside the Slot template call must match the value of mode of the Snippet wrapper template. When defining a manual template, you should change the value of mode automatically generated by XSpLit (rxcas-#) to a unique, user-defined value to avoid conflicts with templates generated by XSpLit.

Suppose for some pages we wanted to override the formatting of the Slot defined in the topic "Defining Slots with Local Formatting" to alternate the color of every other Snippet. Best practice would be:

  1. Define the local formatting as desired, and then use XSpLit to split the HTML.
  2. Copy the code for the Slot template and the Snippet wrapper template from the Format XSL tab either to the top of your HTML source (surround the templates with <!-- begin XSL --> and <!-- end XSL --> comments) or to the rx)Slots.xsl stylesheet (located in the rx_resources/stylesheets/assemblers directory).
  3. Modify the templates by changing the value of mode in the Slot template match and the Snippet wrapper and add the desired programming logic.

Thus, we would modify the example code by adding the following XSL:

<xsl:if test="position() mod 2 != 0">

<xsl:attribute name="class"><xsl:text>bggrey</xsl:text></xsl:attribute>

</xsl:if>

Thus, the Snippet wrapper template would appear as follows:

<xsl:template match="linkurl" mode="AltRowColor">

<tr>

<!-- Apply CSS class to even rows so they can be shaded -->

<xsl:if test="position() mod 2 != 0">

<xsl:attribute name="class"><xsl:text>bggrey</xsl:text></xsl:attribute>

</xsl:if>

<td>&nbsp;</td>

<td>

<xsl:copy-of select="document(Value/@current)/*/body/*">

</td>

<td>&nbsp;</td>

</tr>

</xsl:template>

And the Slot template would be:

<xsl:template match"="rxslot[@template='Related Feature']" mode="rxslot">

<xsl:if test="linkurl">

<table width="100%" border="1" bordercolor="black" cellspacing="0" cellpadding="0">

<tr>

<td class="nav_head">

<div align="center">Related Features</div>

</td>

</tr>

<tr>

<td class="nav_middle">

<table width="100%" border="0" bordercolor="black" cellspacing="0" cellpadding="0">

<!-- start snippet wrapper -->

<!-- This is the Snippet wrapper call --!>

<xsl:apply-templates select="linkurl" mode="AltRowColor"/>

<!-- end snippet wrapper -->

</table>

</td>

</tr>

</table>

<br/>

</xsl:if>

</xsl:template>
