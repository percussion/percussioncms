About XML Fields and DTDs

A Document Type Definition (DTD) defines permissible fields and their hierarchy for an XML document.  DTDs are typically separate files that XML documents refer to.

A powerful feature of XML documents is that you can hierarchically organize fields in a tree structure with different levels of nesting.  Each hierarchical component of a field name is a node.  Each XML field or node is an element within the DTD.  If a field hierarchy is not displayed in a tree, forward slashes in the field name distinguish nodes from element names that make up the field.

Example: person/address/street

  • The person element is the first node of the field.
  • The address element is the second node and is nested under the Person node.
  • The street element is the final part of the field nested under the Address node.

XML field names can begin with a letter or an underscore.  All remaining characters can be any combination of letters, digits, periods, hyphens, and underscores.  Field names are case-sensitive in XML.  The XML field names MyAddress, Myaddress, and myaddress are all different fields because of the different capitalization.

All XML fields in a document must share a common node, the root node.
