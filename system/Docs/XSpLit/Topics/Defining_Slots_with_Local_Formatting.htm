Defining Slots with Local Formatting

In addition to calling the content of the Slot, you generally want to add formatting to the slot.  In most cases, the formatting for the Slot is local to the assembler, but you can also use a remote template.

Defining a Slot with local formatting requires up to three parts:

  • The Slot as a whole.  This portion of the formatting defines the border and background of the Slot, as well as any heading for it.  This formatting does not repeat within the Slot.
  • The Snippet wrapper.  This portion of the formatting defines the immediate formatting around each content item in the slot, such as spacing between each content item.  This formatting repeats with each content item in the Slot.
  • The local content.

The Slot and snippet wrapper formatting are not necessary.  A Slot need only consist of the local content.  However, to add formatting to the Slot, you need to at least define the Slot formatting.  If the local content consists of more than one content item, you will generally want a snippet wrapper for each item.

To mark up a Slot:

  • Use the comments <!-- start slot --> and <!-- end slot --> to indicate the beginning and ending of the Slot.  The format of the comments must be specific:

    <start><whitespace><slot><whitespace>

    <end><whitespace><slot><whitespace>

    Additional whitespace around this pattern is acceptable, as are further comments after the specified formatting.

  • Use the comments <!-- start snippet wrapper --> and <!-- end snippet wrapper --> to indicate the beginning and end of the snippet wrapper.  The format of the comments must be specific:

    <start><whitespace><snippet><whitespace><wrapper><whitespace>

    <end><whitespace><snippet><whitespace><wrapper><whitespace>

    Additional whitespace around this pattern is acceptable, as are further comments after the specified formatting.

  • Use the code specified in Labelling Local Content to define the local content.

For example:

<!-- start slot -->

<table width="100%" border="1" bordercolor="black" cellspacing="0" cellpadding="0">

<tr>

<td class="nav_head">

<div align="center">Related Features</div>

</td>

</tr>

<tr>

<td class="nav_middle">

<table width="100%" border="0" bordercolor="black" cellspacing="0" cellpadding="0">

<!-- start snippet wrapper -->

<tr>

<td>&nbsp;</td>

<td>

<!-- this is the local content call -->

<span slotname="Related Feature">

Snippet example

</span>

</td>

<td>&nbsp;</td>

</tr>

<!-- end snippet wrapper -->

</table>

</td>

</tr>

</table>

<br id="xsplit"/>

<!-- end slot -->

When split, this code would produce the following XSL:

<!--

Code snippet inside the HTML body which calls the slot templates.

There might be multiple slots, so the variable name needs to be unique. It will be enumerated starting at 1.

-->

. . .

<!-- start slot -->

<xsl:variable name="rxslot-2">

<rxslot template="Related Feature">

<xsl:copy-of select="$related/linkurl[@slotname=&apos;Related Feature&apos;]"/>

</rxslot>

</xsl:variable>

<xsl:apply-templates select="$rxslot-2" mode="rxslot"/>

<!-- end slot -->

. . .

<!--

Slot template created from XSpLit.

-->

<xsl:template match="rxslot[@template=&apos;Related Feature&apos;]" mode="rxslot">

    <xsl:if test="linkurl">

      <table width="100%" border="1" bordercolor="black" cellspacing="0" cellpadding="0">

        <tr>

          <td class="nav_head">

            <div align="center">Related Features</div>

          </td>

        </tr>

        <tr>

          <td class="nav_middle">

            <table width="100%" border="0" bordercolor="black" cellspacing="0" cellpadding="0">

              <!--   start snippet wrapper   -->

              <xsl:apply-templates select="linkurl" mode="rxcas-2"/>

              <!--   end snippet wrapper   -->

            </table>

          </td>

        </tr>

      </table>

      <br id="xsplit"/>

    </xsl:if>

  </xsl:template>

<!--

Snippet wrapper template created from XSpLit.

-->

<xsl:template match="linkurl" mode="rxcas-2">

<tr>

<td>&nbsp;</td>

<td>

<xsl:copy-of select="document(Value/@current)/*/body/*">

</td>

<td>&nbsp;</td>

</tr>

</xsl:template>

Note two items in this code that are required for a Slot template. First, the value of mode in the Slot template call itself must be rxslot., demonstrated in the first line of the template:

<xsl:template match="rxslot[@template=&apos;Related Feature&apos;]" mode="rxslot">

Second, the value of mode for the Snippet wrapper and the local content must be the same. In this case, the snippet wrapper includes this line:

<xsl:apply-templates select="linkurl" mode="rxcas-2"/>

Note that the value of mode, rxcas-2, equals the value of mode for the local content call in this line:

<xsl:template match="linkurl" mode="rxcas-2">
