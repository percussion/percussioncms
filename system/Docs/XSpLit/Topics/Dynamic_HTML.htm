Dynamic HTML

Script languages can contain characters such as '<', '&', etc. These characters have a special meaning in XML, and in general, the XSL processor replaces them with an <Escape> character. In order to process these characters correctly, XSpLit encloses scripts in CDATA (character data) sections. This markup ensures that the contents of the CDATA section are read as character data.

For example, suppose you created the following code:

. . .

<script language="javascript">

function test()

{

  var a = 0;

  var b = 1;

  if (a<b && a==0)

  a = b;

}

</script>

. . .

XSpLit would generate the following output:

. . .

<script language="javascript">

<! [CDATA[

function test()

{

  var a = 0;

  var b = 1;

  if (a<b && a==0)

  a = b;

}

]]}

</script>

. . .

This formatting would ensure that the reserved characters (in this case, &&) are read correctly as character data, rather than processed by the XSL processor.
