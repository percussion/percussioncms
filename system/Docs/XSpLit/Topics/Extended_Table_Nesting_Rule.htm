Extended Table Nesting Rule

All other repeatable formats nested within a table, such as a bulleted list, exhaust the shared nodes (from left to right) in the order of outermost to innermost repeatable formatting.

For example, suppose you had the following HTML code:

<table border="1" width="50%">
<tr bgcolor="lightgrey">
<th align="left">Product Name</th>
<th align="left">Product Features</th>
</tr>
<tr>
<td>psx-product/name</td>
<td>
<ul>
<li>psx-product/feature</li>
</ul>
</td>
</tr>
</table>

Rhythmyx processes the outermost table row first. The row repeats once per product. Rhythmyx then processes the nested bulleted list. A bulleted item is repeated once per 'feature.'
