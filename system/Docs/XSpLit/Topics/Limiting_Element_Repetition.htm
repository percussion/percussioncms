Limiting Element Repetition

The repeat behavior of table rows, lists, and form fields depends on the the psx- node hierarchy. Usually, XSpLit requires one node level for each nested repeating element. But in some cases, particularly when using tables to control formatting, you may not create an exact parallel between the node levels and the nesting levels. In this case, the wrong nesting level may repeat.

To ensure that the correct nesting level repeats, use the psx-repeat attribute. The syntax is:

psx-repeat="node1/node2"

The value of psx-repeat is the tree up to the node you want to repeat. In this case, node2 would repeat.

For example, suppose you have three products as input for the following code:

<form method="POST">
<p>
<select name="choices" size="1">
<option selected value="none">Select a product</option>
<option value="psx-product/number">psx-product/name</option>
</select>
<input type="submit" value="Go" name="Go">
</p>
</form>

At design time, the results might look acceptable, but at run time, you would get the following output:

What you really wanted was one field with three options. In order to prevent the repetition at the product node, you would add the psx-repeat attribute to the tag, as in the following code:

<form method="POST">
<p>
<select name="choices" size="1">
<option selected value="none">Select a product</option>
<option value="psx-product/number" psx-repeat="product">psx-product/name</option>
</select>
<input type="submit" value="Go" name="Go">
</p>
</form>
