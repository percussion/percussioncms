Using XSpLit

Before working with XSpLit, identify dynamic content in the HTML page with prefixed field names. After specifying the field names, run XSpLit to create XML, XSL, DTD, and HTML documents.

The following procedure describes the basic steps to transform documents with XSpLit.

  1. Create HTML pages and label the static content to define dynamic content. You can create the HTML using your preferred HTML authoring tool, or create it directly in the XSpLit Source HTML tab.
  2. Open the HTML file (if you did not create it directly in XSpLit). Some ways to open the file include selecting File > Open, clicking the open button , or dragging and dropping the file from Windows Explorer to the XSpLit Source HTML Tab.
  3. (Optional) Select Tools > Options and specify the Identifying tag for HTML markup. For more information, see "Changing the Dynamic Data Prefix".

  4. Enter the Base location for relative URLS.  This field defines the location of stylesheets on your development server so XSpLit find them when testing context-sensitive output.
  5. Select Tools > Split or click to generate DTD, XSL and Sample XML files.
  6. The Content: XML tab is selected automatically after splitting. Add data to the sample XML file.
  7. Select Tools > Merge or click to combine XML and XSL into an HTML document.
  8. Select Tools > Launch Browser or click to view the output file with your browser.
  9. XSpLit displays any errors or processing messages in the output pane at the bottom of the dialog.  You can maximize the output window by clicking the button.  To minimize the output pane, click the button.  To clear the output pane, click the button.

  10. Save your work. You can either:
    • Choose File > Save As to save the document in the current tab format.
    • Choose File > Save All As to open the dialog box with multiple file options. Specify the directory for your saved files, and check the documents you want to save.
