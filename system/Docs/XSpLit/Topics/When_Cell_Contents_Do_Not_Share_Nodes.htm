When Cell Contents Do Not Share Nodes

If you create a table row whose cells contain different fields that do not share any nodes, the row does not repeat. In such a case, you generally cannot tell which occurrence of a node in one cell is related to an occurrence in the other cell.

For example, suppose you have the field names psx-product/name and psx-feature occurring in the following code:

<table border="1" width="50%">
<tr bgcolor="lightgrey">
<th align="left">Product Name</th>
<th align="left">Product Features</th>
</tr>
<tr>
<td>psx-product/name</td>
<td>psx-feature</td>
</tr>
</table>

At runtime, the results would look like the following example. Notice that you cannot tell which feature goes with which product, so it does not make sense to organize the repeated data into rows.

Content: XML

Output: HTML

<products>
<product>
<name>Widget</name>
<name>Gadget</name>
<name>Gizmo</name>
</product>
<feature>It's cheap!</feature>
<feature>It's easy!</feature>
<feature>It's fun!</feature>
<feature>Low maintenance</feature>
</products>
